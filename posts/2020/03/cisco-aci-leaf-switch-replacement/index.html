Cisco ACI - Leaf Switch Replacement

Note: Before following any of the steps below, please take note of the POD ID, Node ID and Node Name of the switch to be replaced. When you unbox the new switch, also note down the serial number of the new switch.

Power on the new switch and connect a console cable to check if it is running in ACI mode or NX-OS mode. If it is running in NX-OS mode, follow the steps documented in Cisco ACI - Converting NX-OS to ACI Boot Mode to convert the switch to ACI mode.

1. Decommission the faulty switch and remove it from the controller

  1. In the ACI GUI, navigate to Fabric > Inventory > Fabric Membership, identify the switch to be replaced.
  2. Right click on the switch to be replaced and select Decommission. The status will show as decommissioned.
  3. Right click on the decommissioned switch and select Remove from Controller. The status will show as decommissioning and this process can take up to 10 minutes.

2. Commission the new switch

Note: Make sure that the new leaf switch is connected to all of the spine switches in the fabric.

  1. In the ACI GUI, navigate to Fabric > Inventory > Fabric Membership > Nodes Pending Registration and look for a new switch that doesn’t have an IP address or node ID assigned. Confirm it is the new switch by verifying that the serial number matches what you noted earlier.
  2. Right click on the new switch to and select Register. Configure the POD ID, Node ID and Node Name. Click Register. The status will go from unknown to discovered and the APIC will assign it an IP address and add it to the Registered Nodes tab.
  3. On the Registered Nodes tab, right click on the new switch and select Commission. The status will go to active.

3. Verify the new switch status

  1. In the ACI GUI, navigate to Fabric > Inventory > Topology to verify that the new switch is now part of the topology.
  2. SSH to the APIC and run the command acidiag fnvread to confirm that the new switch shows up as “active”.

4. Updating your known_hosts file on the APIC

If you try to SSH to the new leaf switch from the APIC and you have connected to the faulty leaf switch previously via SSH, permission will be denied and you will be presented with warnings about “POSSIBLE DNS SPOOFING” and “REMOTE HOST IDENTIFICATION HAS CHANGED”.

This is because the RSA host key has changed for the replacement leaf switch and does not match the RSA host key in known_hosts. To fix this you need to remove the RSA host key from your known_hosts file on the APIC as follows:

apic1# bash
cph-local@apic1:~> ssh-keygen -R <leaf-switch-hostname>
# Host <leaf-switch-hostname> found: line 2
/home/cph-local/.ssh/known_hosts updated.
Original contents retained as /home/cph-local/.ssh/known_hosts.old
cph-local@apic1:~> exit
exit
apic1# ssh <leaf-switch-hostname>
Warning: Permanently added '<leaf-switch-hostname>,100.64.64.64' (RSA) to the list of known hosts.
